**Version:** 1.0
**Date:** December 2024
**Status:** Requirements Specification

---

## Executive Summary

This document defines the UX requirements for redesigning the Jetvision Assistant frontend to align with Avinode's API workflow constraints. The key change is that Avinode's API does not return flight search results directlyâ€”instead, it returns a request ID and deep link, requiring users to complete flight selection within Avinode's marketplace before returning to Jetvision with a TripID.

| Aspect | OLD Workflow | NEW Workflow |
|--------|-------------|--------------|
| Flight Search | Direct results from API | Deep link to Avinode marketplace |
| Flight Selection | Within Jetvision | Within Avinode Web UI |
| Data Return | Immediate flight list | Request ID only, then TripID |
| Quote Collection | Automated in Jetvision | Via Avinode + webhooks |
| User Context Switch | None | Jetvision â†’ Avinode â†’ Jetvision |

### 4.3 Interaction Behavior

**On Click:**
- Load trip's chat thread in main panel
- Highlight selected card with accent border
- Fetch latest trip data if stale (>5 min)

**On Hover:**
- Show subtle elevation shadow
- Display "View Details" tooltip

**Context Menu (Right-click):**
- Copy TripID
- Open in Avinode
- Archive Trip
- Cancel Trip

### 4.4 Sorting/Filtering Requirements

- Last Updated (default)

- Pending Quotes
- Ready for Proposal
- Completed
- Archived

**Search:**
- TripID
- Route (airport codes)
- Date range

### 5.2 Map Integration Requirements

**Map Provider:** Mapbox GL JS or Google Maps

**Required Features:**
- Custom markers for departure/arrival airports
- Great circle arc for flight path
- Automatic zoom to fit both airports with padding
- Airport labels with ICAO codes
- Distance measurement display
- Optional: FBO location markers

**Map Styling:**
- Light mode: Standard terrain with aviation-friendly colors
- Dark mode: Dark theme with contrasting markers
- Minimal UI (hide unnecessary controls)

### 5.3 Image Requirements and Sources

**Aircraft Images:**
| Source | Priority | Fallback |
|--------|----------|----------|
| Avinode API (if provided) | 1 | - |
| Aircraft type stock photos | 2 | Generic category image |
| Placeholder with aircraft silhouette | 3 | Always available |

**Airport Images:**
| Source | Priority | Fallback |
|--------|----------|----------|
| Google Places API | 1 | - |
| Airport database (preloaded) | 2 | Generic airport image |
| Placeholder with location pin | 3 | Always available |

**Image Specifications:**
- Aircraft: 1200x675px (16:9), WebP format, lazy loaded
- Airport thumbnails: 240x160px (3:2), WebP format
- All images should have blur placeholder while loading

### 7.2 Update UI in Real-Time

**Real-time Communication:** Supabase Realtime or Server-Sent Events (SSE)

**UI Update Triggers:**

| Event | UI Update |
|-------|-----------|
| New quote received | Add quote card to chat, update sidebar badge, play notification sound |
| Quote status change | Update status badge, refresh quote card |
| New chat message | Append to chat thread, update "last updated" timestamp |
| Trip status change | Update sidebar card status, show toast notification |

**Implementation:**

```typescript
// Hook: useWebhookUpdates
const useWebhookUpdates = (tripId: string) => {
  useEffect(() => {
    const channel = supabase
      .channel(`trip:${tripId}`)
      .on('postgres_changes', {
        event: 'INSERT',
        schema: 'public',
        table: 'webhook_events',
        filter: `trip_id=eq.${tripId}`
      }, (payload) => {
        handleWebhookEvent(payload.new);
      })
      .subscribe();

    return () => {
      supabase.removeChannel(channel);
    };
  }, [tripId]);
};
```

### 8.3 Active Trip Selection

```typescript
// Actions
type TripAction =
  | { type: 'SELECT_TRIP'; tripId: string }
  | { type: 'LOAD_TRIP'; tripId: string }
  | { type: 'TRIP_LOADED'; trip: Trip }
  | { type: 'ADD_TRIP'; trip: Trip }
  | { type: 'UPDATE_TRIP'; tripId: string; updates: Partial<Trip> }
  | { type: 'REMOVE_TRIP'; tripId: string }
  | { type: 'SET_ERROR'; error: string };

// Reducer
function tripReducer(state: TripState, action: TripAction): TripState {
  switch (action.type) {
    case 'SELECT_TRIP':
      return { ...state, activeTripId: action.tripId };
    case 'LOAD_TRIP':
      return { ...state, isLoadingTrip: true, tripError: null };
    // ... etc
  }
}
```

```sql
CREATE TABLE trips (

  arrival_name VARCHAR(100),

  departure_time TIME,
  passengers INTEGER NOT NULL,

  -- Status
  status VARCHAR(20) DEFAULT 'pending',

  updated_at TIMESTAMPTZ DEFAULT NOW(),
```
